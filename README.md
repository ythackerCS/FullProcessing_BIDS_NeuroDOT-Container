# NeuroDOT Preprocessing Container

## Overview

`Preprocessing_BIDS_NeuroDOT-Container` is a Docker container for running a NeuroDOT preprocessing workflow using Papermill. The container mounts fNIRS subject data from an XNAT scan resource, executes a preprocessing notebook, saves generated figures, and exports the completed notebook for review.

This container was designed to make NeuroDOT preprocessing more reproducible, portable, and accessible through XNAT Container Service.

## Motivation

Functional near-infrared spectroscopy (fNIRS) and diffuse optical tomography (DOT) workflows often require multiple preprocessing, visualization, and quality-control steps. Running these steps manually can make analysis difficult to reproduce across users, projects, and institutions.

This container helps standardize the preprocessing workflow by packaging the required Python environment, NeuroDOT notebook, and XNAT command configuration into a reusable pipeline.

## Key Features

- Runs NeuroDOT preprocessing through a Jupyter notebook executed with Papermill
- Mounts subject-level fNIRS data from XNAT scan resources
- Saves generated figures to an output folder
- Saves the executed notebook for review and reproducibility
- Supports XNAT Container Service integration
- Can be adapted for project-specific fNIRS preprocessing workflows

## Outputs

The container saves outputs back to XNAT or to the mounted output directory, depending on how it is run.

Expected outputs include:

- Executed preprocessing notebook
- Saved preprocessing figures
- Intermediate or processed files generated by the notebook, depending on workflow configuration

Default output folders include:

- `savedFigures/`
- `outputNotebook/`

## Installation and Build

Clone the repository:

```bash
git clone https://github.com/ythackerCS/Preprocessing_BIDS_NeuroDOT-Container.git
cd Preprocessing_BIDS_NeuroDOT-Container
```

Edit the preprocessing notebook if needed:

```text
workspace/neuro_dot/NeuroDOT_PreProcessing_Script_dynamicFilterMode.ipynb
```

Build the Docker container:

```bash
./build.sh
```

If deploying through XNAT, update the XNAT command configuration:

```text
xnat/command.json
```

XNAT Container Service documentation is available here:

```text
https://wiki.xnat.org/container-service/making-your-docker-image-xnat-ready-122978887.html
```

## Running on XNAT

1. Navigate to the relevant subject or session in XNAT.
2. Select **Run Containers**.
3. Choose the NeuroDOT preprocessing container command.
4. Select the scan resource folder containing the fNIRS data.
5. Run the container.
6. Review outputs in the generated resource folders, including saved figures and the executed notebook.

## Running Outside XNAT

This container can also be adapted for local Docker execution by mounting input and output directories manually. Users may need to modify the notebook and upload/output logic depending on whether XNAT-specific functionality is required.

The main preprocessing workflow is defined in:

```text
workspace/neuro_dot/NeuroDOT_PreProcessing_Script_dynamicFilterMode.ipynb
```

If running outside XNAT, remove or modify any XNAT-specific upload steps in the notebook or supporting scripts.

## Customization

Most project-specific changes should be made in:

```text
workspace/neuro_dot/NeuroDOT_PreProcessing_Script_dynamicFilterMode.ipynb
```

Container-level changes can be made in:

```text
Dockerfile.base
```

XNAT command-level changes can be made in:

```text
xnat/command.json
```

This container was designed around MATLAB `.mat`-style fNIRS data inputs and may require adaptation for other data structures.
